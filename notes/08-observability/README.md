# 08 — Observability

Covers monitoring, health checking, and autoscaling.

## Topics
- **Liveness Probes** — restart unhealthy containers automatically
- **Readiness Probes** — gate traffic until the pod is ready
- **Startup Probes** — handle slow-starting containers
- **HorizontalPodAutoscaler (HPA)** — CPU/memory based scaling
- **Metrics Server** — cluster-level resource metrics
- **Events & kubectl top** — real-time resource usage
